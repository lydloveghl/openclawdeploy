# OpenClaw 一键部署与使用手册

这份手册是给 `openclawdeploy` 配套用的。

目标很简单：你把 OpenClaw 装起来，然后知道它怎么接模型、怎么接渠道、怎么查状态、怎么排错。

---

## 1. 这套部署方案的思路

`openclawdeploy` 并不是重写一套 OpenClaw 安装器，而是：

1. **优先复用 OpenClaw 官方安装脚本**
2. 再补上自动化动作：
   - 写 `~/.openclaw/openclaw.json`
   - 配置常见模型认证
   - 安装 Feishu 插件
   - 安装并启动 Gateway

这样做的好处是：

- 更贴近官方支持路径
- 后续升级不容易打架
- 出问题时更容易按官方文档排查

## 2. 平台支持说明

### macOS

推荐，支持较完整。

安装：

```bash
cd ~/openclawdeploy
chmod +x install.sh scripts/deploy.mjs
./install.sh --provider openrouter --api-key <YOUR_OPENROUTER_API_KEY>
```

### Linux

推荐，Gateway 支持成熟。

安装：

```bash
cd ~/openclawdeploy
chmod +x install.sh scripts/deploy.mjs
./install.sh --provider openrouter --api-key <YOUR_OPENROUTER_API_KEY>
```

### Windows

**官方推荐优先走 WSL2**，因为 OpenClaw 在 Linux 运行时更稳。

但本项目也提供原生 PowerShell 入口：

```powershell
cd $HOME\openclawdeploy
powershell -ExecutionPolicy Bypass -File .\install.ps1 -- --provider openrouter --api-key <YOUR_OPENROUTER_API_KEY>
```

如果你是正式生产环境，我建议：

- Windows 本机开发测试可以原生跑
- 真正常驻服务优先：**WSL2 / Linux / macOS**

## 3. 部署完成后，配置文件在哪

OpenClaw 默认读取：

```text
~/.openclaw/openclaw.json
```

这是 **JSON5**，所以允许：

- 注释
- 尾逗号

如果脚本发现已有配置，不会直接覆盖；你需要加 `--force`，它会先自动备份，例如：

```text
~/.openclaw/openclaw.json.bak.2026-03-09T12-00-00.000Z
```

## 5. 接入大模型 API

这套项目内置了三种最常见的认证快捷方式：

- OpenAI
- Anthropic
- OpenRouter

如果你不用这三种，也可以把 `--provider none`，先装好 OpenClaw，后面再手动 `openclaw onboard`。

### 5.1 OpenAI

部署命令：

```bash
./install.sh --provider openai --api-key <OPENAI_API_KEY>
```

默认模型：

```text
openai/gpt-5.4
```

如果要改模型：

```bash
./install.sh --provider openai --api-key <OPENAI_API_KEY> --model openai/gpt-5.4
```

### 5.2 Anthropic

部署命令：

```bash
./install.sh --provider anthropic --api-key <ANTHROPIC_API_KEY>
```

默认模型：

```text
anthropic/claude-opus-4-6
```

### 5.3 OpenRouter

部署命令：

```bash
./install.sh --provider openrouter --api-key <OPENROUTER_API_KEY>
```

默认模型：

```text
openrouter/anthropic/claude-sonnet-4-5
```

这是一个很实用的默认值：便宜、稳、模型选择空间大。

### 5.4 手动走官方 onboard

如果你不想在命令里传 key，也可以安装完成后自己执行：

```bash
openclaw onboard
```

或者按官方推荐方式：

#### OpenAI

```bash
openclaw onboard --openai-api-key "$OPENAI_API_KEY"
```

#### Anthropic

```bash
openclaw onboard --anthropic-api-key "$ANTHROPIC_API_KEY"
```

#### OpenRouter

```bash
openclaw onboard --auth-choice apiKey --token-provider openrouter --token "$OPENROUTER_API_KEY"
```

## 6. 接入 Telegram

OpenClaw 当前对 Telegram 是原生支持的。

### 6.1 先创建 Bot

1. 打开 Telegram
2. 找 **@BotFather**
3. 执行 `/newbot`
4. 保存 Bot Token

### 6.2 部署时直接带上参数

```bash
./install.sh \
  --provider openrouter \
  --api-key <OPENROUTER_API_KEY> \
  --with-telegram \
  --telegram-bot-token <TELEGRAM_BOT_TOKEN>
```

### 6.3 生成的核心配置

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "123:abc",
      dmPolicy: "pairing",
      groups: {
        "*": { requireMention: true },
      },
    },
  },
}
```

### 6.4 启动后怎么让私聊生效

```bash
openclaw gateway start
openclaw pairing list telegram
openclaw pairing approve telegram <CODE>
```

### 6.5 群里不回复，通常看这几个点

- Bot 没加到群
- BotFather 的隐私模式没处理
- 群里默认要求 `@bot`
- pairing / allowlist 没配置好
- `api.telegram.org` 网络不通

### 6.6 如果你要让群里默认不需要 @

部署时加：

```bash
--telegram-require-mention false
```

或者手动改配置：

```json5
{
  channels: {
    telegram: {
      groups: {
        "*": { requireMention: false },
      },
    },
  },
}
```

## 7. 接入 Feishu / Lark

OpenClaw 对 Feishu 的支持来自 **单独插件**，不是主程序内置。

### 7.1 插件安装

本项目在你启用 `--with-feishu` 时，会自动执行：

```bash
openclaw plugins install @openclaw/feishu
```

如果你想自己装：

```bash
openclaw plugins install @openclaw/feishu
```

### 7.2 去飞书开放平台创建应用

你需要：

1. 打开 Feishu Open Platform
2. 创建企业自建应用
3. 打开 **Bot** 能力
4. 记录：
   - App ID
   - App Secret
5. 事件订阅选择 **长连接（WebSocket）**
6. 开启 `im.message.receive_v1`
7. 发布应用

### 7.3 部署时带参数

```bash
./install.sh \
  --provider openrouter \
  --api-key <OPENROUTER_API_KEY> \
  --with-feishu \
  --feishu-app-id cli_xxx \
  --feishu-app-secret yyy \
  --feishu-bot-name 公司助手
```

如果是国际版 Lark：

```bash
./install.sh \
  --provider openrouter \
  --api-key <OPENROUTER_API_KEY> \
  --with-feishu \
  --feishu-domain lark \
  --feishu-app-id cli_xxx \
  --feishu-app-secret yyy
```

### 7.4 生成的核心配置

```json5
{
  channels: {
    feishu: {
      enabled: true,
      domain: "feishu",
      dmPolicy: "pairing",
      accounts: {
        default: {
          appId: "cli_xxx",
          appSecret: "xxx",
          botName: "OpenClaw",
        },
      },
    },
  },
}
```

### 7.5 启动后检查

```bash
openclaw gateway status
openclaw logs --follow
```

### 7.6 飞书常见问题

#### 看起来配置好了，但收不到消息

通常是下面几种：

- 插件没安装
- Gateway 没启动
- 事件订阅没保存成功
- 没发布应用版本
- 权限范围不够

#### 飞书国际版不通

检查是不是应该把：

```json5
channels.feishu.domain = "lark"
```

## 8. 接入 QQ

### 8.1 先说实话

根据当前本机 OpenClaw 官方文档，**没有原生 QQ channel 文档，也没有内置 QQ 官方接入说明**。

所以这套项目不会骗你说：

> “QQ 已经像 Telegram 一样开箱即用。”

不是。当前更现实的做法是：

- **NapCat / OneBot / go-cqhttp 一类 QQ 适配层**
- 再做一个桥，把 QQ 消息转成 OpenClaw 可以消费的输入
- 把 OpenClaw 输出再回写到 QQ

也就是说，QQ 目前更像一个 **桥接工程**，不是这版脚本能直接原生开关的渠道。

### 8.2 你应该怎么理解这个边界

- **Feishu / Telegram**：这版可以直接配置
- **QQ**：这版提供手册和落地路线，但不假装原生支持

详细说明看：

- [QQ接入说明.md](./QQ接入说明.md)

## 10. 故障排查顺序

如果部署完成后不好使，我建议按这个顺序查：

### 第一步：CLI 在不在

```bash
openclaw --version
```

### 第二步：总状态

```bash
openclaw status
```

### 第三步：Gateway 状态

```bash
openclaw gateway status
```

### 第四步：重启服务

```bash
openclaw gateway restart
```

### 第五步：看日志

```bash
openclaw logs --follow
```

### 第六步：doctor

```bash
openclaw doctor --non-interactive
```

### 第七步：检查配置文件

```bash
cat ~/.openclaw/openclaw.json
```

重点看：

- model 是否写对
- token / appId / appSecret 是否为空
- telegram / feishu 是否 enabled
- 是否 accidentally 覆盖掉旧配置

## 11. 升级建议

后续如果你要继续把这套东西做得更像真正的“生产部署工具”，建议优先补这几件：

1. `.env` 文件支持
2. 读取并合并现有 `openclaw.json`，而不是整文件覆盖
3. Docker Compose 版本
4. Linux systemd / macOS launchd 的更细粒度检查
5. QQ OneBot bridge 落地
6. 渠道健康检查（telegram token test / feishu app test）

---

## 12. 最后一句

如果你只是想最快跑起来：

- 模型先用 **OpenRouter**
- 渠道先接 **Telegram 或 Feishu**
- QQ 先别强行一步到位

这样成功率最高，也最接近 OpenClaw 官方支持路径。
