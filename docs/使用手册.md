# OpenClaw 一键部署与使用手册

这份手册是给 `openclawdeploy` 配套用的。

目标很简单：你把 OpenClaw 装起来，然后知道它怎么接模型、怎么接渠道、怎么配 skills、怎么查状态、怎么排错。

现在这套项目已经同时支持两种形态：

1. **终端模式**
2. **桌面安装器模式（目标产物：dmg / exe）**

如果你的目标是“像普通安装程序一样双击打开”，重点看桌面安装器模式。

---

## 1. 这套部署方案的思路

`openclawdeploy` 并不是重写一套 OpenClaw 安装器，而是：

1. **优先复用 OpenClaw 官方安装脚本**
2. 在它上面补一层 **交互向导 / 图形安装界面**
3. 再补上自动化动作：
   - 写 `~/.openclaw/openclaw.json`
   - 配置常见模型认证
   - 支持跳过模型接入
   - 支持自定义大模型 provider
   - 安装 Feishu 插件
   - 安装并启动 Gateway
   - 配置 skills
4. 最外层再包成 **桌面安装程序**

这样做的好处是：

- 更贴近官方支持路径
- 后续升级不容易打架
- 出问题时更容易按官方文档排查
- 最终用户体验更像普通安装器

## 2. 平台支持说明

### macOS

推荐，可打包 `dmg`。

### Linux

可打包 `AppImage`。

### Windows

目标产物为 `exe`（NSIS 安装器）。

> 但如果你要的是 OpenClaw 本体长期稳定运行，OpenClaw 官方依然更推荐 **WSL2 / Linux 运行时**。这不影响安装器本身做成 exe。

## 3. 桌面安装器怎么启动

### 3.1 开发启动

在项目目录执行：

```bash
npm install
npm run desktop
```

这会直接打开一个桌面窗口，不会跳浏览器。

### 3.2 打包 macOS dmg

```bash
npm install
npm run package:dmg
```

默认产物目录：

```text
release/
```

### 3.3 打包 Windows exe

更推荐在 Windows 机器上执行：

```powershell
npm install
npm run package:exe
```

### 3.4 打包 Linux AppImage

```bash
npm install
npm run package:desktop
```

  - OpenRouter

## 5. 跳过大模型接入

如果你只想先把 OpenClaw 装起来，不想现在就接模型，完全可以。

### 终端方式

```bash
./install.sh --provider none
```

### 桌面方式

在安装器界面把模型提供方选成：

- **先跳过**

这样会：

- 安装 OpenClaw
- 写基础配置
- 不配置默认模型
- 不执行模型认证

## 6. 自定义大模型 provider

如果你有自己的代理、企业网关，或者兼容 OpenAI / Anthropic 协议的第三方模型服务，可以这样接。

### 终端方式

```bash
./install.sh \
  --provider custom \
  --custom-provider-id myproxy \
  --custom-api openai-completions \
  --custom-base-url https://example.com/v1 \
  --custom-model-id gpt-4.1 \
  --custom-model-name "My Proxy GPT-4.1" \
  --custom-api-key sk-xxx
```

### 桌面方式

在安装器里选择：

- 模型提供方 → **自定义 provider**

然后填写：

- Provider ID
- API Adapter
- Base URL
- Model ID
- Model Name
- API Key

### 支持的 adapter

- `openai-completions`
- `openai-responses`
- `anthropic-messages`
- `google-generative-ai`

## 7. 配置文件在哪

OpenClaw 默认读取：

```text
~/.openclaw/openclaw.json
```

这是 **JSON5**，所以允许：

- 注释
- 尾逗号

如果脚本发现已有配置：

- 图形界面会让你勾选“覆盖已有配置（会自动备份）”
- 终端模式下需要你显式加 `--force`

覆盖前会自动备份，例如：

```text
~/.openclaw/openclaw.json.bak.2026-03-09T12-00-00.000Z
```

## 9. 接入大模型 API

这套项目内置了三种最常见的认证快捷方式：

- OpenAI
- Anthropic
- OpenRouter

如果你不用这三种，也可以：

- `--provider none`，先装好 OpenClaw
- 或者 `--provider custom`，写入自定义 provider
- 或者安装完成后再手动 `openclaw onboard`

### 9.1 OpenAI

```bash
openclaw onboard --openai-api-key "$OPENAI_API_KEY"
```

### 9.2 Anthropic

```bash
openclaw onboard --anthropic-api-key "$ANTHROPIC_API_KEY"
```

### 9.3 OpenRouter

```bash
openclaw onboard --auth-choice apiKey --token-provider openrouter --token "$OPENROUTER_API_KEY"
```

## 10. 接入 Telegram

OpenClaw 当前对 Telegram 是原生支持的。

### 10.1 先创建 Bot

1. 打开 Telegram
2. 找 **@BotFather**
3. 执行 `/newbot`
4. 保存 Bot Token

### 10.2 配置方式

无论是桌面安装器还是终端模式，本质都会写出类似配置：

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "123:abc",
      dmPolicy: "pairing",
      groups: {
        "*": { requireMention: true },
      },
    },
  },
}
```

### 10.3 启动后怎么让私聊生效

```bash
openclaw gateway start
openclaw pairing list telegram
openclaw pairing approve telegram <CODE>
```

## 11. 接入 Feishu / Lark

OpenClaw 对 Feishu 的支持来自 **单独插件**。

### 11.1 插件安装

启用 Feishu 时，安装器会自动执行：

```bash
openclaw plugins install @openclaw/feishu
```

### 11.2 去飞书开放平台创建应用

你需要：

1. 打开 Feishu Open Platform
2. 创建企业自建应用
3. 打开 **Bot** 能力
4. 记录 App ID / App Secret
5. 事件订阅选择 **长连接（WebSocket）**
6. 开启 `im.message.receive_v1`
7. 发布应用

### 11.3 生成的核心配置

```json5
{
  channels: {
    feishu: {
      enabled: true,
      domain: "feishu",
      dmPolicy: "pairing",
      accounts: {
        default: {
          appId: "cli_xxx",
          appSecret: "xxx",
          botName: "OpenClaw",
        },
      },
    },
  },
}
```

## 12. 配置 Skills

现在 `openclawdeploy` 已经支持把 skills 配置直接写进 `~/.openclaw/openclaw.json`。

### 12.1 这版支持写哪些字段

- `skills.allowBundled`
- `skills.load.extraDirs`
- `skills.load.watch`
- `skills.load.watchDebounceMs`
- `skills.install.preferBrew`
- `skills.install.nodeManager`
- `skills.entries.<skillKey>.enabled`
- `skills.entries.<skillKey>.apiKey`
- `skills.entries.<skillKey>.env`
- `skills.entries.<skillKey>.config`

### 12.2 生效时机

一般来说：

- **新会话**最稳妥
- watcher 开着时，下一次 agent turn 也可能自动刷新

如果你改完感觉没生效，先：

```bash
openclaw gateway restart
```

然后新开一个会话再试。

## 13. 接入 QQ

### 13.1 先说实话

根据当前本机 OpenClaw 官方文档，**没有原生 QQ channel 文档，也没有内置 QQ 官方接入说明**。

所以这套项目不会骗你说：

> “QQ 已经像 Telegram 一样开箱即用。”

当前更现实的做法是：

- NapCat / OneBot / go-cqhttp 这类 QQ 适配层
- 再做一个 bridge，把 QQ 消息转给 OpenClaw
- 再把 OpenClaw 输出回写到 QQ

详细说明看：

- [QQ接入说明.md](./QQ接入说明.md)

## 14. 典型部署方案推荐

### 最省事

- 平台：macOS / Linux
- 模型：OpenRouter
- 渠道：Telegram 或 Feishu
- skills：按需配，不要一次开太多

### 企业 / 内网代理

- 平台：Linux 服务器
- 模型：OpenAI / Anthropic / OpenRouter / 自定义 provider
- 渠道：Feishu
- skills：统一放共享目录，再通过 `skills.load.extraDirs` 加载

## 15. 故障排查顺序

### 第一步：CLI 在不在

```bash
openclaw --version
```

### 第二步：总状态

```bash
openclaw status
```

### 第三步：Gateway 状态

```bash
openclaw gateway status
```

### 第四步：重启服务

```bash
openclaw gateway restart
```

### 第五步：看日志

```bash
openclaw logs --follow
```

### 第六步：doctor

```bash
openclaw doctor --non-interactive
```

### 第七步：检查配置文件

```bash
cat ~/.openclaw/openclaw.json
```

重点看：

- model 是否写对
- custom provider 的 baseUrl / api / model id 是否写对
- token / appId / appSecret 是否为空
- telegram / feishu 是否 enabled
- skills 配置有没有写错结构
- 是否不小心覆盖掉了旧配置
